# Fork Status (PSMDB gh-pages)

Jekyll site for **Percona Server for MongoDB** fork status: shows divergence from upstream (e.g. MongoDB `master`), a table of unmerged commits with merge-pick hints, filters, and PR status.

## Run locally

### Prerequisites

Install Ruby and Bundler. On Ubuntu/Debian:

```bash
sudo apt-get install ruby-full build-essential
```

### Setup and run

1. Configure Bundler to install gems locally (avoids permission issues):

   ```bash
   bundle config set --local path 'vendor/bundle'
   ```

2. Install dependencies:

   ```bash
   bundle install
   ```

   > **Note:** Do NOT use `sudo bundle install` — this will install gems as root and cause permission errors for your regular user.

3. Start the Jekyll server:

   ```bash
   bundle exec jekyll serve
   ```

4. Open [http://localhost:4000](http://localhost:4000).

## Data source

Branch data is loaded from **`_data/branches/`**. For example:

- **`_data/branches/master.json`** - divergence, last merged commit, first unmerged commit, list of unmerged commits, merge picks (with strategy and details), repo URLs, etc.

This JSON is **generated by mergai fork status**. This repo does not generate it; you need to run mergai (or copy its output) and place `master.json` (and optionally other branch files) into `_data/branches/` so the site has data to show.

## Structure

- **`index.md`** — main page; uses `site.data.branches.master` and includes summary grid, filter controls, and commits table.
- **`_includes/`** — Liquid partials (commits table, filter controls, summary grid, number formatting).
- **`assets/css/style.scss`** — theme overrides and layout (Jekyll theme: `jekyll-theme-midnight`).
- **`assets/js/`** — filter, PR status (GitHub API), copy-SHA with accessibility (aria-live).
